HydrateMe
App Portfolio

ZAKIRAH BINTI ZULKIFLEE @ LAURCAL  ·  A209855  ·  Software Engineering

A mobile application built with Kotlin and Jetpack Compose to help users track their daily water intake, set hydration goals, and stay connected with a community challenge — powered by Room, Firebase Firestore, Retrofit, and live location services.

💧 SDG 6: Clean Water and Sanitation — because access to safe water and conscious hydration habits are fundamental to human health and well-being.

Laboratory Work

Labs 1 – 5

Lab 01

UI Design with Jetpack Compose

Built foundational UI components using Jetpack Compose — understanding composables, layouts, and the declarative UI paradigm in Android development.

This lab introduced me to Jetpack Compose and how it differs from XML-based layouts. I learned how to structure composables and manage state, which formed the backbone of my later projects.

Lab 02

Interaction & State Management

Explored user interaction handling, state hoisting, and event-driven UI updates using remember and mutableStateOf in Compose.

Understanding state management was a turning point — seeing how UI automatically reacts to data changes made me appreciate the reactive model. This directly influenced how I managed hydration state in HydrateMe.

Lab 03

Material Design 3

Applied Material Design 3 components — cards, buttons, typography scales, colour schemes, and theming — to build polished, accessible interfaces.

Material Design 3 gave my UIs a professional, consistent look. I applied card elevations, colour schemes and typography in HydrateMe to make the app feel cohesive and user-friendly.

Lab 04

Navigation with NavController

Implemented multi-screen navigation using Jetpack Navigation Compose, including a bottom navigation bar, route management, and back stack handling.

Navigation Compose simplified multi-screen flow drastically. Building the NavHost and bottom navigation bar in this lab directly translated into the five-tab navigation structure of HydrateMe.

Lab 05

Room Database Persistence

Integrated Room — Android's local SQLite abstraction — to persist data using Entities, DAOs, and a RoomDatabase class with coroutine-based Flow queries.

Room was the most challenging lab but the most rewarding. Implementing DAOs, Flows, and schema migrations prepared me to build HydrateMe's water_logs and goal tables with confidence.

Application Projects

Projects 1 & 2

Project 1

HydrateMe — Core Features

Problem Statement: Many people fail to drink enough water daily due to a lack of reminders and awareness. HydrateMe provides a simple interface to log, track, and visualise daily water intake with goal-setting features.

Project 1 taught me how to connect Navigation, ViewModel, and UI state into a cohesive app. I implemented bottom navigation, a log screen, stats screen, and profile screen — cementing my understanding of clean architecture in Android.

Project 2

HydrateMe — Full Stack + Cloud

Problem Statement: Extending the core app with cloud sync, API integration, and community features — users can now compare their hydration with others, check weather-based hydration advice, and persist all data locally and in the cloud simultaneously.

Project 2 was the most technically demanding work I have done. Integrating Room, Firebase Firestore, Retrofit weather API, and GPS location simultaneously — while managing state across a ViewModel — gave me real-world mobile development experience. The Community Challenge leaderboard was the most satisfying feature to see working in real-time.

Learning Journey

My Reflection

Starting from Lab 1 where I barely understood composables, to deploying a fully functional cloud-connected Android application in Project 2, this course has been a transformative journey. Each lab built on the last — UI, interaction, Material Design, navigation, and persistence — forming the complete foundation for HydrateMe. The most significant growth came from learning to connect multiple technologies simultaneously: Room for local persistence, Firebase Firestore for real-time cloud sync, Retrofit for weather data, and GPS sensors for location — all wired through a single ViewModel with clean state management. I now feel confident approaching real-world Android development challenges and understand how thoughtful architecture makes an app maintainable, scalable, and genuinely useful.
